OSOlogic Get started

Build cool things on OSOLogic

One idea powers everything: every value is a tag in the in-memory hub (osodb), backed by the database. Read it or command it from SQL, REST, OPC-UA, MQTT or an AI agent — and every HMI, script and dashboard uses the same tags.

These examples target a running instance (e.g. the sandbox at http://localhost:8080). Copy, paste, tweak.

1 Control a tag — three ways, same result

Turn the demo pump on. Pick your protocol:

# REST — from any language
curl http://localhost:8080/var/hass.switch.pump
curl -X PUT http://localhost:8080/var/hass.switch.pump -d '{"value":1}'
-- SQL — the database is the plant
UPDATE tags SET required_value = 1 WHERE id = 'hass.switch.pump';
SELECT id, value, units FROM tags;
# OPC-UA — point any client at the server
opc.tcp://localhost:4840/osologic/   # ns: urn:osologic:sandbox

Read & write tags from any language — PHP · Java · C# · VB · Rust · Python · R · MATLAB · Excel →

4 Script it

Read and drive tags from a script (shell, Python, Node… in the Scripting console).

Open Scripting → 
import urllib.request, json
base = "http://localhost:8080"
lvl = json.load(urllib.request.urlopen(base+"/var/hass.sensor.tank_level"))["value"]
if lvl < 30:
    urllib.request.urlopen(urllib.request.Request(base+"/var/hass.switch.pump",
        data=b'{"value":1}', method="PUT"))   # top up
