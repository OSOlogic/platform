Database · osodb source of truth

Pick the backend osodb persists to. The real-time path is the in-memory cache, so this is a deployment choice, not a performance one — the same tags work on every engine.

osodb is the cache and owns the real-time loop, so the engine only has to persist and carry the SQL control surface. That's why PostgreSQL or SQLite work as the source of truth without MariaDB's MEMORY tables, and why a microcontroller can run the MCU engine (embedded store + MariaDB emulation). See the osodb adapters.
