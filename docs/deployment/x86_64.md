# Deploying OSOLogic on x86_64 (industrial PC / VM)

**© 2026 Roig Borrell S.L. · Ibercomp S.L.** · Part of [OSOLogic](https://github.com/OSOlogic/platform) · AGPL-3.0

> **Status: 🔜 Planned.** Official x86_64 images are on the roadmap. The runtime and installer
> run on standard x86_64 today — install onto an existing Linux as shown below. This is the
> easiest way to **evaluate OSOLogic in a VM**.

Targets for x86_64 builds live in
[`os-dist/osologic-linux/targets/x86_64`](../../os-dist/osologic-linux/targets/x86_64).

---

## Good for

- **Industrial PCs** and panel PCs used as soft-PLC / edge controllers.
- **Virtual machines** — a quick, throwaway way to try the whole stack.
- **Servers** running SCADA/Historian-scale workloads.

## Install on an existing Linux

1. Start from a 64-bit Debian / Ubuntu (bare metal or VM).
2. Put the platform under the `oso` home:
   ```bash
   sudo useradd -m -s /bin/bash oso 2>/dev/null || true
   sudo -u oso git clone https://github.com/OSOlogic/platform.git /home/oso/PLC_OsoLogic
   ```
3. Run the installer:
   ```bash
   sudo oso-setup            # guided wizard (ncurses + plain-text fallback)
   # or, for full control:
   cd /home/oso/PLC_OsoLogic/packaging && sudo ./install_OsoLogic.sh
   ```
   Choose **Advanced → source: already on this board** so it uses your clone.

4. Verify — Service Manager at `https://<host-ip>:8080`. The installer steps are identical to
   [orange-pi.md](orange-pi.md).

## Real-time notes

For deterministic control on x86_64, run a **PREEMPT_RT** kernel (see
[`core/osokernel`](../../core/)) and pin the runtime to isolated cores. In a VM, real-time
guarantees are best-effort — great for functional testing, not for hard-real-time control.

## I/O on x86_64

Without onboard GPIO/SPI, connect field I/O over the **gateways** — Modbus TCP/RTU,
EtherNet/IP, PROFINET, OPC-UA or CANopen (USB/PCIe adapters). See [`gateways/`](../../gateways/).
